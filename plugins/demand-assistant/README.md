# demand-assistant

> 需求分析助手 — Claude Code 插件
>
> 输入在线文档 URL、聊天截图或需求描述,通过多轮交互澄清需求边界,自动生成结构化需求文档(Markdown)。

## 功能简介

- **多源输入**:支持在线文档 URL(网页抓取)、本地图片(多模态识别)、纯文本三种原始材料
- **边界澄清**:从 6 个维度评估材料完整度,先列出待确认问题,等用户回复后再生成文档,避免"猜需求"
- **结构化输出**:严格按内置模板填充,支持 `[待补充]` / `[推断]` / `[需确认]` 三种标注规范
- **本地落盘**:文档自动保存到当前工作目录,文件名包含日期与主题关键词

## 安装

将本目录作为 Claude Code 插件加载即可。插件清单位于 `.claude-plugin/plugin.json`。

依赖外部 MCP 工具 `mcp__web_reader__webReader`(用于抓取 URL 内容);如未配置,纯文本与本地图片场景仍可正常工作。

## 使用方式

### 方式一:斜杠命令(推荐)

```text
/analyze-demand <URL | 图片路径 | 需求描述文本>
```

示例:

```text
/analyze-demand https://example.com/doc/123
/analyze-demand /mnt/d/screenshots/chat.png
/analyze-demand 老板说要做一个库存预警功能,缺货时自动通知运营
```

参数为空时,助手会主动询问原始材料。

### 方式二:调用 Agent

当对话中涉及"分析需求 / 写需求文档 / 整理需求"等意图时,Claude 会自动委派给 `requirement-analyst` 子代理处理。

## 执行流程

1. **获取材料** — 按输入类型分别使用 `webReader` / `Read` / 直接文本
2. **边界澄清** — 列出【必须确认】+【建议确认】问题(共 3-8 个),等待用户回复
3. **生成文档** — 按 `skills/analyze-demand/template.md` 填充
4. **保存文件** — 落盘为 `需求文档-{YYYY-MM-DD}-{主题关键词}.md`

## 需求边界澄清的 6 个维度

| 维度 | 关注点 |
|------|--------|
| 需求来源与干系人 | 提出方、使用角色、影响范围 |
| 业务背景与痛点 | 现状、核心问题、发生频率 |
| 功能范围与边界 | 包含 / 不包含 / 系统联动 |
| 业务规则与约束 | 公式、阈值、异常兜底、权限 |
| 数据依赖 | 表、接口、时效性、数据量 |
| 验收标准 | 完成判定、性能/兼容性、回归影响 |

## 文档模板结构

```text
一、需求背景(必填)
   需求来源 / 适用部门 / 调研人群 / 业务背景 / 实现目标
二、详细描述(必填)
   功能名称 / 所在路径 / 实现步骤(规则/触发节点/计算规则/约束/数据来源)
三、补充说明(选填)
   影响范围 / 风险点 / 上下游依赖 / 待确认事项
```

完整模板见 `skills/analyze-demand/template.md`,真实示例见 `需求模板.md`。

## 标注规范

| 标注 | 含义 |
|------|------|
| `[待补充]` | 原始材料完全未提及 |
| `[推断]` | 根据上下文合理推测 |
| `[需确认]` | 存在矛盾或不确定 |
